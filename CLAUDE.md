## What This Project Is

Hunter is a browser-based Boids predator game hosted at hunter.job-joseph.com.
A FastAPI backend on the Pi serves both the React frontend (static files) and
the leaderboard API. Single repo, single service, single subdomain.

## Pi Deployment Pattern

FastAPI serves the built React frontend as static files — no Nginx, no Vercel.
The Cloudflare Tunnel routes hunter.job-joseph.com → localhost:8013.

### Deploy sequence (after any change):

**Frontend change:**
```bash
cd ~/projects/hunter/frontend
npm run build
sudo systemctl restart hunter
```

**Backend change:**
```bash
sudo systemctl restart hunter
```

**First deploy only:**
```bash
cd ~/projects/hunter/backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
sudo cp /home/jcube/projects/hunter/backend/hunter.service /etc/systemd/system/hunter.service
sudo systemctl daemon-reload
sudo systemctl enable hunter
sudo systemctl start hunter
```

## Key Constraints — Read Before Touching These

- **StaticFiles must be mounted LAST in main.py** — after all /api/* routes.
  Mounting first causes StaticFiles to intercept API calls and return 404s.
- **leaderboard.db is gitignored** — created automatically on first run via
  SQLite. Never commit it.
- **frontend/dist/ is gitignored** — always built on the Pi during deploy.
  Never commit built output.
- **All Boids tuning constants live in frontend/src/constants/boids.js** —
  never hardcode values inline. Tuning happens in one place only.
- **No CORS config needed** — frontend and API share the same origin
  (hunter.job-joseph.com). Do not add CORSMiddleware.
- **No authentication** — this is a public game. No Cloudflare Access.
- **window.__hunter dev hook** — exists in App.jsx, gated behind
  `import.meta.env.DEV`. Exposes fishRef, predatorRef, scoreRef,
  timeLeftRef, gameState for Playwright sessions. Never appears in
  production builds.
- **Process cleanup** — when stopping dev servers, kill only the specific
  PIDs on ports 5173 and 8013. Never run a broad process sweep — other
  Pi services (ports 8001–8012) are systemd-managed and will auto-recover
  but a sweep is careless.
